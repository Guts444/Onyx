## 2024-03-24 - Convert Array Includes to Set Has in List Rendering
**Learning:** Found an O(N^2) bottleneck when rendering `ChannelShelf` because `Array.includes()` was being used inside a list mapping over channels for both favorites and recents. Even if N is only ~10,000, calling `Array.includes` on a large array inside another loop creates an O(N * M) rendering overhead that blocks the main thread.
**Action:** Always use `Set.has()` (O(1)) instead of `Array.includes()` when filtering or checking membership inside large map/filter loops in React components. Also added `useMemo` for filtering list items in `App.tsx` (`visibleChannels`) to avoid allocating and re-filtering arrays on every render (especially when renders are triggered by frequent changes like player volume updates).
## 2024-02-23 - Map Lookup and Set Allocation Optimization in EPG Channel Indexing
**Learning:** Checking Map contents via `.has()` followed by `.get()` and conditional `.set()` creates duplicate hash lookups. Modifying Map checking to rely on `undefined` checks after a single `.get()` saves measurable time. In addition, avoiding `Set` instantiation to build variant array strings drastically reduces CPU/Memory overhead.
**Action:** Always favor `.get(key) === undefined` instead of `.has(key)` to avoid dual lookups, and avoid `Set` and spread allocations inside hot tight loops if simple array checks logic is achievable.
