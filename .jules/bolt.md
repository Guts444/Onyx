## 2025-03-09 - Memoized dictionaries vs Lazy lookup hooks
**Learning:** Constructing a massive mapping dictionary inside `useMemo` is expensive when iterating over a very large array (e.g. 50,000 items), blocking the main thread. Switching to a `useCallback` that performs a lazy lookup when a component actually needs the data for rendering drastically reduces up-front CPU cost and memory allocations.
**Action:** When deriving state for large lists where only a small subset is rendered or accessed at any given time, prefer lazy evaluation via parameterized getters (`useCallback`) instead of full pre-computation (`useMemo`).
