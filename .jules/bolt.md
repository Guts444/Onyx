## 2024-05-24 - JavaScript String Iteration Performance and Safety
**Learning:** Replacing a `for...of` string iteration with an index-based `charCodeAt()` loop can slightly improve raw performance by avoiding iterator overhead, but it alters the algorithm's output for non-BMP characters (like emojis), causing subtle functional regressions in deterministic operations like hashing.
**Action:** When optimizing repeated, deterministic computations where accuracy for all character sets is essential, prefer a memoization cache over altering the core iteration algorithm if it risks changing the output.
