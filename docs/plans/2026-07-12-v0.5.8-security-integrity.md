# Onyx v0.5.8 Security and Integrity Implementation Plan

> **For Hermes:** Use subagent-driven-development and strict TDD task-by-task.

**Goal:** Eliminate credential persistence and unsafe remote playback targets, make identifiers and persistence crash-safe at 50k+ channel scale, close stale-operation/network failure modes, and produce a verified v0.5.8 Windows release candidate.

**Architecture:** Persist credential-free channel descriptors and reconstruct Xtream URLs only at playback time from OS-keyring-backed source state. Separate immutable playlist cache from mutable selection metadata, version/migrate persisted state, serialize writes per key, and use atomic temp-file replacement with backups. Make channel identity source-aware and collision-resistant while preserving/migrating user references.

**Tech Stack:** Tauri 2, Rust, React 19, TypeScript, Node test runner, libmpv, Windows Credential Manager.

---

## Acceptance invariants

1. No persisted app-state, legacy localStorage, logs, error text, or release artifact may contain an Xtream password.
2. Remote M3U/Xtream entries cannot open drive-letter paths, UNC shares, or `file:` URLs; trusted local imports retain explicit local playback support.
3. Channel IDs are deterministic, source-aware, and at least 128-bit; favorites/recents/selection/manual EPG mappings survive migration where old→new channels can be matched.
4. State/cache writes are ordered per key, atomic, backed up, bounded, schema-validated, and recoverable after corruption.
5. Deleted/edited sources and guides cannot be resurrected by stale requests; imports have cancellation/overall deadlines.
6. Valid gzip EPG is decoded exactly once; malformed programmes are skipped with diagnostics rather than rejecting an entire guide.
7. Production and dev builds use separate app identifiers.
8. `npm test`, type-check, frontend build, Rust tests, fmt, clippy, npm audit, cargo audit, release packaging, and isolated visual smoke tests pass.

## Task 1 — Security utility tests and source-aware channel identity

**Files:** modify `src/features/playlist/channelFactory.ts`, its tests, channel/domain types and call sites.

- RED: tests for remote rejection, trusted-local allowance, deterministic 128-bit source-aware IDs, no collisions in a large fixture.
- GREEN: introduce import trust/source context and stable WebCrypto-independent digest suitable for synchronous parsing.
- Migrate references after import by matching legacy channel identity to new channels.

## Task 2 — Credential-free snapshots and runtime URL materialization

**Files:** create playlist snapshot sanitizer/materializer modules; modify `App.tsx`, Xtream import, player boundary, persistence tests.

- RED: persisted Xtream/M3U snapshots contain no credentials; runtime URL can be reconstructed only with in-memory/keyring secret; error strings redact all credential URL forms.
- GREEN: persist stream descriptors/redacted URLs; hydrate runtime playback URL immediately before libmpv command; remove legacy credential-bearing snapshot/localStorage after safe migration/refresh.

## Task 3 — Atomic versioned persistence and recovery

**Files:** `src-tauri/src/lib.rs`, `src/hooks/usePersistentState.ts`, Rust tests.

- RED: interrupted/corrupt primary recovers backup; oversized state rejected; invalid schema quarantined; writes for one key remain ordered.
- GREEN: temp-write + sync + atomic replace/rename + backup; backend limits; per-key serialization; frontend schema/version parsers and visible degradation state.
- Split playlist cache from selected-channel/session metadata so selection does not rewrite the library.

## Task 4 — EPG integrity and network lifecycle

**Files:** `src-tauri/src/epg.rs`, `src/App.tsx`, EPG tests.

- RED: corrupt cache self-recovers, gzip decodes once, malformed programme is skipped, stale refresh cannot repopulate deleted source, total deadlines expire.
- GREEN: per-cache backup/quarantine, magic-byte gzip detection, tolerant parser diagnostics, operation generations/cancellation, total request deadlines, `spawn_blocking` for heavy parsing/serialization where appropriate.

## Task 5 — Source races and credential hydration resilience

**Files:** `src/App.tsx`, source services/components, tests.

- RED: later source selection wins; deleted source cannot become active; one keyring failure does not block other profiles.
- GREEN: operation IDs/AbortControllers, source-existence commit checks, disable/cancel conflicting UI operations, `Promise.allSettled` hydration.

## Task 6 — Dependency, CSP, and development isolation hardening

- Upgrade Vite/esbuild/Babel, quick-xml, rustls stack, Tauri-compatible dependencies until audits are clean or remaining target-inapplicable advisories are documented.
- Narrow CSP and libmpv permissions where plugin support permits.
- Add tracked isolated dev config/launcher and prerequisite validation; pin package manager, Node/Rust versions, and native DLL provenance/checksums.

**Files:** `.github/workflows/ci.yml`, package scripts, tests.
